options(mostly use default, REQUIRES_NEW)
* REQUIRED: default
  - if previous transaction exist, use it
  - else, create new transaction
* REQUIRES_NEW: create new transaction

* SUPPORTS
  - if previous transaction exist, use it
  - else, do nothing
* NOT_SUPPORTED: not use transaction
* MANDATORY: must use previous transaction
  - if previous transaction not exist, use
  - else, throw exception(IllegalTransactionStateException)
* NEVER: never use transaction
  - if previous transaction exist, throw exception(IllegalTransactionStateException)
  - else, proceed
* NESTED:
  - if previous transaction exist, create nested transaction.
    - nested transaction affects only current transaction.
      - nested rollback, but outer transaction commit
    - outer transaction affects nested transaction.
      - if outer rollback, nested rollback
  - else, create new transaction
  - caution: need JDBC savepoint. check DB driver support?

* transaction propagation and option
- isolation, timeout, readOnly can be set when new transaction start. but not participate in previous transaction.
- ex. REQUIRED, REQUIRES_NEW
